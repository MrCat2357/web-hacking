### 6.2.1 schemata


Uma tabela importante é a `schemata`, que guarda informações sobre todas as bases de dados. Para verificar as informações presentes nela, usamos

`select * from schemata`

```mysql
MariaDB [information_schema]> select * from schemata;
+--------------+--------------------+----------------------------+------------------------+----------+----------------+
| CATALOG_NAME | SCHEMA_NAME        | DEFAULT_CHARACTER_SET_NAME | DEFAULT_COLLATION_NAME | SQL_PATH | SCHEMA_COMMENT |
+--------------+--------------------+----------------------------+------------------------+----------+----------------+
| def          | information_schema | utf8mb3                    | utf8mb3_general_ci     | NULL     |                |
| def          | mysql              | utf8mb4                    | utf8mb4_uca1400_ai_ci  | NULL     |                |
| def          | performance_schema | utf8mb3                    | utf8mb3_general_ci     | NULL     |                |
| def          | desec              | utf8mb4                    | utf8mb4_uca1400_ai_ci  | NULL     |                |
| def          | sys                | utf8mb3                    | utf8mb3_general_ci     | NULL     |                |
+--------------+--------------------+----------------------------+------------------------+----------+----------------+
```

Essa tabela é importante na hora de um ataque pelo motivo de muitas vezes estarmos às cegas na fase da exploração de algum injection, ela pode nos dar um norte no que diz respeito às estruturas das nossas bases de dados.

Para direcionar a pesquisa à base que queremos, no caso a desec, podemos

`select table_name from tables where table_schema='desec';`

```mysql
MariaDB [information_schema]> select table_name from tables where table_schema='desec';
+------------+
| table_name |
+------------+
| usuarios   |
+------------+
```

Podemos fazer uma busca mais específica também por colunas

```mysql
MariaDB [information_schema]> select column_name from columns where table_schema="desec";
+-------------+
| column_name |
+-------------+
| id          |
| login       |
| senha       |
+-------------+
```

Isso é importante pra entendermos a estrutura da base de dados, que vai nos ajudar à fazer o dump na hora de um mysqli, por exemplo.
Vamos agora até a database desec:

`show databases;`

`use desec;`

``select login,senha from usuarios;`

```mysql
MariaDB [desec]> select login,senha from usuarios;
+--------+-----------+
| login  | senha     |
+--------+-----------+
```

Podemos inclusive concatenar as informações por meio do comando `concat`

`select concat(login, ':', senha) from usuarios;`

```mysql
MariaDB [desec]> select concat(login, ':', senha) from usuarios;
+---------------------------+
| concat(login, ':', senha) |
+---------------------------+
| admin:admin               |
| catulo:123                |
| axel:123456               |
| desec:webmaster           |
| web:webmaster             |
| admin:master              |
+---------------------------+
```

Isso pode ser útil em ataques de SQLi pelo fato de as vezes termos apenas um campo para receber as informações
